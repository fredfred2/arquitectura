There are many things going on here: f:viewParam must appear within f:metadata.
f:metadata must appear directly under the view root.
That is, you can not define it within h:head or h:body.
The name attribute of f:viewParam defines the name of the URL parameter.
This value is set to the property identified by the value attribute.
That means, in our case, the id URL parameter will be set as the lookupId property of MainController.
The f:event tag is used to have the action handler method invoked when a GET request arrives for result.xml.
